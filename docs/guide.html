MASTERN Inbound Agent

인바운드 자동화 에이전트

Outlook·Teams로 수신되는 업무 메일을 자동으로 분류하고 Jira 티켓을 생성하는 그룹웨어 자동화 시스템입니다.

Python 3.11+ FastAPI Microsoft Graph API Ollama LLM Jira Windows
개요

프로젝트 개요

이 에이전트는 그룹웨어(Outlook/Teams)로 수신되는 업무 요청 메일을 자동으로 수집·분류하고 Jira 티켓으로 연결하는 자동화 파이프라인입니다.

📥
자동 메일 수집
Microsoft Graph API를 통해 Outlook 메일을 5분마다 자동 수집합니다.
🤖
AI 자동 분류
규칙 기반 + Ollama LLM으로 메일 의도를 긴급/문의/작업요청 등으로 분류합니다.
📋
Jira 티켓 생성
분류 결과에 따라 Jira 이슈(Bug/Task/Story)를 자동 또는 수동으로 생성합니다.
📊
대시보드 & 리포트
웹 대시보드에서 수신 현황 조회, 기간별 리포트, 차트를 확인할 수 있습니다.

설치

설치 & 실행

요구 사항

  • Python 3.11 이상
  • Windows 10/11 (pywin32 의존성)
  • Ollama 로컬 설치 및 모델 다운로드 (llama3.2 권장)
  • Jira 계정 및 API 토큰 (Jira 연동 사용 시)
  • Azure AD 앱 등록 (Microsoft Graph 인증용)

최초 설치 (1회만)

1
프로젝트 폴더로 이동
cd D:\inbound_gw_agent
2
가상환경 생성 & 패키지 설치
python -m venv .venv
.venv\Scripts\pip install -r requirements.txt
3
.env 파일 생성
프로젝트 루트에 .env 파일을 만들고 아래 환경 변수 설정 섹션을 참고해 값을 입력하세요.
4
서버 실행
.venv\Scripts\python.exe -m inbound_gw_agent

서버 재시작 (한 줄 명령어)

$p = (netstat -ano | Select-String ":5000\s.*LISTENING" | Select-Object -First 1).ToString().Trim() -split "\s+" | Select-Object -Last 1; if ($p) { Stop-Process -Id $p -Force }; Start-Sleep -Seconds 1; cd D:\inbound_gw_agent; .venv\Scripts\python.exe -m inbound_gw_agent
⚠️
.env 파일 수정 후에는 반드시 서버를 재시작해야 변경 내용이 적용됩니다.

연동

Webhook (Power Automate)

Power Automate를 이용해 Outlook/Teams 메시지를 실시간으로 전달할 수 있습니다.

엔드포인트

POST http://localhost:5000/webhook/message
Content-Type: application/json

{
  "source": "outlook",
  "sender": "user@company.com",
  "subject": "메일 제목",
  "body": "메일 본문",
  "received_at": "2026-05-18T10:00:00Z"
}

Outlook 플로우 설정

1
트리거: 새 이메일이 도착하면 (V3)
2
HTTP 액션 추가
메서드: POST / URI: ngrok URL + /webhook/message
3
본문 설정
{
  "source": "outlook",
  "sender": "@{triggerBody()?['from']?['emailAddress']?['address']}",
  "subject": "@{triggerBody()?['subject']}",
  "body": "@{triggerBody()?['body']?['content']}",
  "received_at": "@{triggerBody()?['receivedDateTime']}"
}
연동

ngrok 외부 노출

Power Automate가 로컬 서버에 접근할 수 있도록 ngrok으로 외부 URL을 발급합니다.

1
ngrok 설치
winget install ngrok.ngrok
2
별도 터미널에서 실행
ngrok http 5000
3
발급된 URL 사용
https://xxxx.ngrok-free.app 형태의 URL을 Power Automate HTTP 액션 URI에 입력합니다.
⚠️
ngrok은 서버가 실행 중일 때만 동작합니다. 컴퓨터 재시작 시 URL이 변경되므로 Power Automate 설정도 함께 업데이트해야 합니다.

FAQ

자주 묻는 질문

❓ 서버가 시작되지 않습니다

포트 5000이 이미 사용 중인지 확인하세요. PowerShell에서 다음 명령으로 확인할 수 있습니다:

netstat -ano | Select-String ":5000"

❓ 메일이 수집되지 않습니다

  • AZURE_CLIENT_ID가 올바르게 설정되었는지 확인
  • .token_cache.bin이 존재하는지 확인 (없으면 재인증 필요)
  • Microsoft 계정 인증이 만료된 경우 .token_cache.bin 삭제 후 재실행

❓ Jira 티켓이 생성되지 않습니다

  • JIRA_ENABLED=true 설정 확인
  • JIRA_API_TOKEN이 유효한지 확인 (만료 시 재발급)
  • JIRA_PROJECT_KEY가 실제 프로젝트 키와 일치하는지 확인
  • 대시보드 ⚙ 설정 → Jira 연결 정보 다시 저장 후 서버 재시작

❓ 담당자가 다른 사람으로 설정됩니다

JIRA_ACCOUNT_ID에 올바른 값을 설정해야 합니다.
Jira Cloud: https://yourdomain.atlassian.net/rest/api/3/myself에 접속해 accountId 값 복사
Jira Server: 본인의 Jira 로그인 ID 입력

❓ Ollama 모델을 변경하고 싶습니다

# Ollama 모델 다운로드
ollama pull gemma3

# .env 수정
OLLAMA_MODEL=gemma3

.env 수정 후 서버를 재시작하세요.

❓ 중복 메일이 수집됩니다

각 메일은 메시지 ID 기반으로 중복 처리됩니다. 동일 메일이 반복 수집되는 경우 state.db 파일의 처리 이력을 확인하세요.

⚠️
문제가 지속될 경우 터미널의 로그 출력을 확인하세요. 에러 내용을 함께 첨부하면 빠르게 해결할 수 있습니다.
